# Chapter 9: Thai Text Samples / บทที่ 9: ตัวอย่างข้อความภาษาไทย {#ch09}

This chapter provides extensive Thai text samples to verify proper word wrapping, line breaking, and text rendering in the generated DOCX output.

บทนี้รวบรวมตัวอย่างข้อความภาษาไทยที่มีความยาวหลากหลายรูปแบบ เพื่อตรวจสอบการตัดคำ การตัดบรรทัด และการแสดงผลข้อความในไฟล์ DOCX ที่สร้างขึ้น

## Long Paragraph / ย่อหน้ายาว {#ch09-long-paragraph}

ภาษาไทยเป็นภาษาที่มีความซับซ้อนในเรื่องของการตัดคำเนื่องจากไม่มีการใช้ช่องว่างระหว่างคำเหมือนภาษาอังกฤษหรือภาษาตะวันตกอื่นๆ ทำให้การพัฒนาระบบตัดบรรทัดอัตโนมัติสำหรับภาษาไทยนั้นมีความท้าทายเป็นอย่างมาก โปรแกรมคอมพิวเตอร์จำเป็นต้องใช้พจนานุกรมหรืออัลกอริทึมพิเศษในการระบุขอบเขตของคำแต่ละคำเพื่อให้สามารถตัดบรรทัดได้อย่างถูกต้อง การแสดงผลข้อความภาษาไทยบนหน้าจอคอมพิวเตอร์หรืออุปกรณ์พกพานั้นต้องคำนึงถึงปัจจัยหลายประการ ไม่ว่าจะเป็นขนาดของตัวอักษร ความกว้างของพื้นที่แสดงผล และความสามารถในการรองรับสระและวรรณยุกต์ที่อยู่เหนือและใต้พยัญชนะ ระบบปฏิบัติการสมัยใหม่ส่วนใหญ่สามารถจัดการกับการแสดงผลภาษาไทยได้ดีขึ้นมากเมื่อเทียบกับในอดีต แต่ก็ยังคงมีปัญหาบางประการที่ต้องได้รับการแก้ไข โดยเฉพาะอย่างยิ่งในเรื่องของการตัดคำที่ถูกต้องตามหลักภาษาศาสตร์

## Mixed Thai-English Content / เนื้อหาผสมไทย-อังกฤษ {#ch09-mixed-content}

การพัฒนา software ในปัจจุบันนั้นต้องอาศัยเครื่องมือหลายตัว เช่น IDE (Integrated Development Environment) อย่าง Visual Studio Code หรือ IntelliJ IDEA ในการเขียน code นอกจากนี้ยังต้องใช้ version control system เช่น Git เพื่อจัดการกับ source code ของ project ต่างๆ อีกด้วย การทำงานร่วมกันในทีมนั้นยังต้องอาศัย platform อย่าง GitHub หรือ GitLab ในการทำ code review และ continuous integration

เทคโนโลยี cloud computing ได้เปลี่ยนแปลงวิธีการ deploy แอปพลิเคชันอย่างมาก ไม่ว่าจะเป็น Amazon Web Services (AWS), Microsoft Azure หรือ Google Cloud Platform (GCP) ล้วนมี service ที่หลากหลายให้เลือกใช้ ตั้งแต่ virtual machine, container orchestration ด้วย Kubernetes ไปจนถึง serverless function อย่าง AWS Lambda หรือ Azure Functions ซึ่งช่วยให้ developer สามารถ focus กับ business logic ได้โดยไม่ต้องกังวลเรื่อง infrastructure

## Technical Thai Text / ข้อความเทคนิคภาษาไทย {#ch09-technical}

### การเขียนโปรแกรมด้วยภาษา Rust

ภาษา Rust เป็นภาษาโปรแกรมระดับระบบที่ถูกออกแบบมาเพื่อความปลอดภัยของหน่วยความจำ ความเร็วในการทำงาน และการทำงานแบบขนาน (concurrency) โดยไม่จำเป็นต้องใช้ garbage collector ระบบ ownership และ borrowing ของ Rust ช่วยป้องกัน bug ที่เกี่ยวกับหน่วยความจำได้ตั้งแต่ขั้นตอน compile time ทำให้โปรแกรมที่เขียนด้วย Rust มีความน่าเชื่อถือสูงและมีประสิทธิภาพเทียบเท่า C/C++ แต่มีความปลอดภัยมากกว่า

คุณสมบัติเด่นของ Rust มีดังนี้:

- **ระบบ Ownership** — ทุกค่าใน Rust มี owner เพียงหนึ่งเดียว เมื่อ owner ออกนอก scope ค่านั้นจะถูก drop โดยอัตโนมัติ ซึ่งช่วยป้องกัน memory leak ได้อย่างมีประสิทธิภาพ
- **ระบบ Borrowing** — สามารถยืมค่า (borrow) ได้ทั้งแบบ immutable reference (&T) และ mutable reference (&mut T) แต่ไม่สามารถมี mutable reference มากกว่าหนึ่งตัวในเวลาเดียวกัน
- **Pattern Matching** — รองรับการจับคู่รูปแบบที่ซับซ้อนผ่าน match expression ซึ่งช่วยให้การจัดการกับ enum variants และโครงสร้างข้อมูลทำได้อย่างสะอาดและชัดเจน
- **Cargo Package Manager** — เครื่องมือจัดการ package ที่ทรงพลัง รองรับการ build, test, benchmark และ publish crate ไปยัง crates.io ได้อย่างสะดวก
- **Zero-Cost Abstractions** — การใช้ abstract ในระดับสูงไม่ส่งผลกระทบต่อประสิทธิภาพการทำงาน โค้ดที่เขียนด้วย generic และ trait จะถูก monomorphize ในขั้นตอน compilation

### การจัดการเอกสาร DOCX

รูปแบบไฟล์ DOCX เป็นมาตรฐาน Office Open XML (OOXML) ที่กำหนดโดย ECMA-376 และ ISO/IEC 29500 ภายในไฟล์ DOCX จะเป็นไฟล์ ZIP ที่ประกอบด้วยไฟล์ XML หลายไฟล์ รวมถึง document.xml สำหรับเนื้อหาหลัก styles.xml สำหรับรูปแบบ settings.xml สำหรับการตั้งค่า และ fontTable.xml สำหรับตารางฟอนต์ การทำความเข้าใจโครงสร้างภายในของ DOCX เป็นสิ่งจำเป็นสำหรับการพัฒนาเครื่องมือที่สร้างหรือแก้ไขไฟล์ DOCX โดยตรง

## Formatted Thai Text / ข้อความไทยที่มีการจัดรูปแบบ {#ch09-formatted}

การเขียนเอกสารที่ดีนั้นจำเป็นต้องใช้การจัดรูปแบบที่เหมาะสม ตัวอย่างเช่น **ข้อความตัวหนา** ใช้สำหรับเน้นความสำคัญ *ข้อความตัวเอียง* ใช้สำหรับคำศัพท์เฉพาะ ~~ข้อความขีดทับ~~ ใช้สำหรับเนื้อหาที่ถูกแก้ไข และ `โค้ดอินไลน์` ใช้สำหรับแสดงชื่อคำสั่งหรือชื่อไฟล์ การผสมผสานรูปแบบเหล่านี้ช่วยให้เอกสารมีความชัดเจนและอ่านง่ายขึ้น

นอกจากนี้ยังสามารถใช้หลายรูปแบบร่วมกันได้ เช่น ***ข้อความตัวหนาและตัวเอียง*** หรือ **ข้อความตัวหนาที่มี `โค้ด` อยู่ภายใน** ซึ่งเป็นประโยชน์เมื่อต้องการเน้นทั้งความสำคัญและแสดงว่าเป็นคำสั่งทางเทคนิค

## Thai Numbered List / รายการลำดับเลขภาษาไทย {#ch09-numbered-list}

1. ขั้นตอนแรกคือการติดตั้ง Rust toolchain โดยใช้คำสั่ง `curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh` ซึ่งจะติดตั้ง rustup, rustc และ cargo ให้อัตโนมัติ
2. จากนั้นให้ clone repository ของ md2docx จาก GitHub มายังเครื่องของคุณ โดยใช้คำสั่ง `git clone` ตามด้วย URL ของ repository
3. เข้าไปยังไดเรกทอรีของโปรเจกต์แล้วรันคำสั่ง `cargo build --release` เพื่อ compile โปรเจกต์ในโหมด release ซึ่งจะให้ประสิทธิภาพสูงสุด
4. ทดสอบการทำงานโดยรันคำสั่ง `cargo test` เพื่อตรวจสอบว่า test case ทั้งหมดผ่านเรียบร้อย
5. หากต้องการติดตั้งให้ใช้งานได้ทั่วระบบ ให้รันคำสั่ง `cargo install --path .` ซึ่งจะ copy ไฟล์ binary ไปยัง `~/.cargo/bin/`

## Conclusion / สรุป {#ch09-conclusion}

ตัวอย่างข้อความในบทนี้แสดงให้เห็นความสามารถของ md2docx ในการจัดการกับข้อความภาษาไทยในรูปแบบต่างๆ ทั้งย่อหน้ายาว ข้อความผสมไทย-อังกฤษ ข้อความเทคนิค การจัดรูปแบบ ตาราง และรายการ ระบบ Complex Script (`<w:cs/>`) ช่วยให้ Microsoft Word สามารถตัดคำภาษาไทยได้อย่างถูกต้องโดยใช้พจนานุกรมตัดคำภายใน ทำให้เอกสารที่สร้างจาก md2docx มีคุณภาพเทียบเท่ากับเอกสารที่พิมพ์โดยตรงใน Microsoft Word
