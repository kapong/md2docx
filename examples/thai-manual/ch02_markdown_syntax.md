# บทที่ 2 รูปแบบการเขียน Markdown {#ch02}

บทนี้จะอธิบายรูปแบบการเขียน Markdown ทั้งหมดที่ md2docx รองรับ พร้อมตัวอย่างภาษาไทย เพื่อให้คุณสามารถสร้างเอกสารที่สวยงามและมีประสิทธิภาพ

## ภาพรวม Markdown

**Markdown** เป็นภาษามาร์กอัปที่ออกแบบมาให้เขียนง่าย อ่านง่าย และแปลงเป็น HTML หรือรูปแบบอื่นๆ ได้ md2docx รองรับ Markdown มาตรฐาน (CommonMark) พร้อมส่วนขยายเพิ่มเติม

### ข้อดีของ Markdown

- **เขียนง่าย** - ใช้สัญลักษณ์ที่เรียบง่าย เช่น `#` สำหรับหัวข้อ
- **อ่านง่าย** - แม้ไม่แปลงก็ยังอ่านเข้าใจได้
- **เก็บใน Git** - เป็น plain text จัดการ version control ได้ง่าย
- **รวดเร็ว** - เขียนได้เร็วกว่าใช้ Word โดยตรง
- **ยูนิเวอร์แซล** - แปลงเป็น HTML, PDF, DOCX ได้

## หัวข้อ (Headings)

หัวข้อใช้สัญลักษณ์ `#` นำหน้าข้อความ จำนวน `#` บ่งบอกระดับของหัวข้อ

### รูปแบบพื้นฐาน

```markdown
# หัวข้อระดับ 1 (Heading 1)
## หัวข้อระดับ 2 (Heading 2)
### หัวข้อระดับ 3 (Heading 3)
#### หัวข้อระดับ 4 (Heading 4)
##### หัวข้อระดับ 5 (Heading 5)
###### หัวข้อระดับ 6 (Heading 6)
```

### ตัวอย่างการใช้งาน

```markdown
# คู่มือการใช้งานระบบ

เอกสารนี้อธิบายวิธีการใช้งานระบบครั้งแรก

## การลงทะเบียน

ขั้นตอนแรกคือการลงทะเบียนบัญชีผู้ใช้

### ขั้นตอนการลงทะเบียน

1. เข้าไปที่หน้าเว็บไซต์
2. คลิกปุ่ม "ลงทะเบียน"
3. กรอกข้อมูลส่วนตัว

#### ข้อมูลที่ต้องกรอก

- ชื่อ-นามสกุล
- อีเมล
- รหัสผ่าน
```

### หัวข้อพร้อม ID

คุณสามารถเพิ่ม ID ให้กับหัวข้อเพื่อใช้ในการอ้างอิง:

```markdown
# บทนำ {#introduction}
## การติดตั้ง {#installation}
### การติดตั้งบน Windows {#install-windows}
```

จากนั้นสามารถอ้างอิงได้:

```markdown
ดูรายละเอียดที่ [บทนำ](#introduction)
หรือ [การติดตั้งบน Windows](#install-windows)
```

### หัวข้อพร้อมข้อความเพิ่มเติม

```markdown
# หัวข้อหลัก ## ข้อความเพิ่มเติม
## หัวข้อรอง ## ข้อความเพิ่มเติม
```

> **หมายเหตุ:** md2docx จะใช้หัวข้อระดับ 1 สำหรับชื่อบท และหัวข้อระดับ 2-3 สำหรับส่วนย่อย

## ย่อหน้า (Paragraphs)

ย่อหน้าคือข้อความธรรมดาที่ไม่มีการจัดรูปแบบพิเศษ เขียนโดยพิมพ์ข้อความแล้วกด Enter 2 ครั้งเพื่อขึ้นบรรทัดใหม่

### การเขียนย่อหน้า

```markdown
นี่คือย่อหน้าแรก ย่อหน้าประกอบด้วยข้อความธรรมดา
ที่อ่านง่ายและเข้าใจได้

นี่คือย่อหน้าที่สอง มีการเว้นบรรทัดว่างหนึ่งบรรทัด
ระหว่างย่อหน้าแรกและย่อหน้าที่สอง

นี่คือย่อหน้าที่สาม คุณสามารถเขียนย่อหน้าได้มากเท่าที่ต้องการ
เพื่ออธิบายเนื้อหาอย่างละเอียด
```

### การขึ้นบรรทัดใหม่ในย่อหน้าเดียวกัน

หากต้องการขึ้นบรรทัดใหม่โดยไม่เริ่มย่อหน้าใหม่ ให้ใช้ 2 ช่องว่าง ที่ท้ายบรรทัด:

```markdown
นี่คือบรรทัดแรก
นี่คือบรรทัดที่สอง (อยู่ในย่อหน้าเดียวกัน)
นี่คือบรรทัดที่สาม
```

หรือใช้ `<br>`:

```markdown
นี่คือบรรทัดแรก<br>
นี่คือบรรทัดที่สอง<br>
นี่คือบรรทัดที่สาม
```

### ตัวอย่างย่อหน้าภาษาไทย

```markdown
ยินดีต้อนรับสู่คู่มือการใช้งานระบบของเรา
เอกสารนี้จะแนะนำวิธีการใช้งานระบบทั้งหมด
ตั้งแต่การลงทะเบียน การใช้งาน ไปจนถึงการแก้ไขปัญหา

ระบบของเราออกแบบมาให้ใช้งานง่าย
มีความปลอดภัยสูง และรองรับภาษาไทยอย่างสมบูรณ์
คุณสามารถเริ่มต้นใช้งานได้ทันทีหลังจากลงทะเบียน
```

## การจัดรูปแบบข้อความ (Text Formatting)

Markdown รองรับการจัดรูปแบบข้อความหลายแบบ เพื่อเน้นข้อความที่สำคัญ

### ตัวหนา (Bold)

ใช้ `**` หรือ `__` คร่อมข้อความ:

```markdown
นี่คือข้อความ **ตัวหนา**
นี่ก็คือข้อความ __ตัวหนา__ เช่นกัน
```

ผลลัพธ์:
- นี่คือข้อความ **ตัวหนา**
- นี่ก็คือข้อความ __ตัวหนา__ เช่นกัน

### ตัวเอียง (Italic)

ใช้ `*` หรือ `_` คร่อมข้อความ:

```markdown
นี่คือข้อความ *ตัวเอียง*
นี่ก็คือข้อความ _ตัวเอียง_ เช่นกัน
```

ผลลัพธ์:
- นี่คือข้อความ *ตัวเอียง*
- นี่ก็คือข้อความ _ตัวเอียง_ เช่นกัน

### ตัวหนาและเอียง (Bold + Italic)

ใช้ `***` หรือ `___` คร่อมข้อความ:

```markdown
นี่คือข้อความ ***ตัวหนาและเอียง***
นี่ก็คือข้อความ ___ตัวหนาและเอียง___ เช่นกัน
```

ผลลัพธ์:
- นี่คือข้อความ ***ตัวหนาและเอียง***
- นี่ก็คือข้อความ ___ตัวหนาและเอียง___ เช่นกัน

### ขีดฆ่า (Strikethrough)

ใช้ `~~` คร่อมข้อความ:

```markdown
นี่คือข้อความ ~~ที่ถูกขีดฆ่า~~
```

ผลลัพธ์:
- นี่คือข้อความ ~~ที่ถูกขีดฆ่า~~

### โค้ดแบบอินไลน์ (Inline Code)

ใช้ `` ` `` คร่อมข้อความ:

```markdown
ใช้คำสั่ง `print()` เพื่อแสดงผล
ตัวแปร `username` จะเก็บชื่อผู้ใช้
```

ผลลัพธ์:
- ใช้คำสั่ง `print()` เพื่อแสดงผล
- ตัวแปร `username` จะเก็บชื่อผู้ใช้

### ตัวอย่างการใช้งานร่วมกัน

```markdown
# คำแนะนำสำคัญ

**ข้อความสำคัญ:** คุณต้อง *อ่าน* ส่วนนี้อย่างละเอียด

~~ข้อความเดิม~~ ถูกแก้ไขเป็น **ข้อความใหม่**

ใช้ฟังก์ชัน `calculate_total()` เพื่อคำนวณยอดรวม
```

### ตัวอย่างภาษาไทย

```markdown
**ข้อความสำคัญ:** โปรดอ่านคู่มือนี้ก่อนเริ่มใช้งาน

*คำเตือน:* การกระทำที่ผิดอาจทำให้ข้อมูลสูญหาย

~~ราคาเดิม 1,000 บาท~~ **ราคาพิเศษ 500 บาท**

ใช้คำสั่ง `npm install` เพื่อติดตั้งแพ็กเกจ
```

## รายการ (Lists)

Markdown รองรับทั้งรายการแบบไม่เรียงลำดับ (Unordered Lists) และรายการแบบเรียงลำดับ (Ordered Lists)

### รายการแบบไม่เรียงลำดับ (Unordered Lists)

ใช้ `-`, `*`, หรือ `+` นำหน้ารายการ:

```markdown
- รายการแรก
- รายการที่สอง
- รายการที่สาม

* รายการแรก
* รายการที่สอง
* รายการที่สาม

+ รายการแรก
+ รายการที่สอง
+ รายการที่สาม
```

ผลลัพธ์:
- รายการแรก
- รายการที่สอง
- รายการที่สาม

### รายการแบบเรียงลำดับ (Ordered Lists)

ใช้ตัวเลขตามด้วย `.` นำหน้ารายการ:

```markdown
1. ขั้นตอนแรก
2. ขั้นตอนที่สอง
3. ขั้นตอนที่สาม
```

ผลลัพธ์:
1. ขั้นตอนแรก
2. ขั้นตอนที่สอง
3. ขั้นตอนที่สาม

> **เคล็ดลับ:** คุณสามารถใช้ตัวเลขผิดก็ได้ Markdown จะจัดเรียงให้อัตโนมัติ:

```markdown
1. รายการแรก
1. รายการที่สอง
1. รายการที่สาม
```

ผลลัพธ์จะเป็น:
1. รายการแรก
2. รายการที่สอง
3. รายการที่สาม

### รายการซ้อน (Nested Lists)

เว้นระยะห่าง 2-4 ช่องว่าง หรือ 1 tab เพื่อสร้างรายการซ้อน:

```markdown
1. หัวข้อหลัก
   - รายการย่อย 1
   - รายการย่อย 2
     - รายการย่อยซ้อน
2. หัวข้อหลักที่สอง
   1. รายการย่อยแบบเรียงลำดับ
   2. รายการย่อยแบบเรียงลำดับ
3. หัวข้อหลักที่สาม
```

ผลลัพธ์:
1. หัวข้อหลัก
   - รายการย่อย 1
   - รายการย่อย 2
     - รายการย่อยซ้อน
2. หัวข้อหลักที่สอง
   1. รายการย่อยแบบเรียงลำดับ
   2. รายการย่อยแบบเรียงลำดับ
3. หัวข้อหลักที่สาม

### ตัวอย่างรายการภาษาไทย

```markdown
# สิ่งที่ต้องเตรียม

ก่อนเริ่มใช้งานระบบ คุณต้องเตรียมสิ่งต่อไปนี้:

- คอมพิวเตอร์ที่เชื่อมต่ออินเทอร์เน็ต
- บัญชีอีเมลที่ใช้งานได้
- เอกสารประจำตัว (สำหรับการยืนยันตัวตน)

## ขั้นตอนการลงทะเบียน

1. เข้าไปที่เว็บไซต์ของเรา
2. คลิกปุ่ม "ลงทะเบียน" ที่มุมขวาบน
3. กรอกข้อมูลในแบบฟอร์ม
   - ชื่อ-นามสกุล
   - อีเมล
   - รหัสผ่าน
4. คลิกปุ่ม "ส่งข้อมูล"
5. ตรวจสอบอีเมลเพื่อยืนยันบัญชี
```

### รายการพร้อมคำอธิบาย (Task Lists)

ใช้ `[ ]` สำหรับรายการที่ยังไม่เสร็จ และ `[x]` สำหรับรายการที่เสร็จแล้ว:

```markdown
- [x] ติดตั้งโปรแกรม
- [x] สร้างบัญชีผู้ใช้
- [ ] ตั้งค่าระบบ
- [ ] ทดสอบการทำงาน
```

ผลลัพธ์:
- [x] ติดตั้งโปรแกรม
- [x] สร้างบัญชีผู้ใช้
- [ ] ตั้งค่าระบบ
- [ ] ทดสอบการทำงาน

## ลิงก์ (Links)

### ลิงก์พื้นฐาน

รูปแบบ: `[ข้อความแสดง](URL)`

```markdown
เยี่ยมชมเว็บไซต์ของเราที่ [md2docx](https://example.com)
ดูเอกสารเพิ่มเติมที่ [คู่มือ](https://example.com/docs)
```

### ลิงก์พร้อม Title

รูปแบบ: `[ข้อความแสดง](URL "Title")`

```markdown
คลิก [ที่นี่](https://example.com "เว็บไซต์หลัก") เพื่อไปยังหน้าหลัก
```

### ลิงก์แบบอ้างอิง (Reference Links)

กำหนดลิงก์ไว้ที่ท้ายเอกสาร:

```markdown
เยี่ยมชม [เว็บไซต์][1] หรือ [คู่มือ][2] ของเรา

[1]: https://example.com
[2]: https://example.com/docs
```

### ลิงก์ไปยังส่วนอื่นในเอกสาร

ใช้ `#` ตามด้วย ID ของหัวข้อ:

```markdown
ดูรายละเอียดที่ [บทนำ](#introduction)
หรือ [การติดตั้ง](#installation)
```

### อีเมลลิงก์

```markdown
ติดต่อเราที่ [support@example.com](mailto:support@example.com)
```

### ตัวอย่างลิงก์ภาษาไทย

```markdown
# แหล่งข้อมูลเพิ่มเติม

สำหรับข้อมูลเพิ่มเติม คุณสามารถดูได้ที่:

- [เว็บไซต์หลัก](https://example.com)
- [คู่มือการใช้งาน](https://example.com/docs)
- [ฟอรัมสนทนา](https://forum.example.com)
- [ช่องทางติดต่อ](https://example.com/contact)

หากมีข้อสงสัย สามารถติดต่อเราได้ที่ [support@example.com](mailto:support@example.com)
```

## ตาราง (Tables)

### ตารางพื้นฐาน

ใช้ `|` คั่นคอลัมน์ และ `-` สำหรับส่วนหัว:

```markdown
| ชื่อ | อายุ | อาชีพ |
|-------|-------|--------|
| สมชาย | 25 | วิศวกร |
| วิภา | 30 | นักการตลาด |
| กิตติ | 28 | นักพัฒนา |
```

ผลลัพธ์:

| ชื่อ | อายุ | อาชีพ |
|-------|-------|--------|
| สมชาย | 25 | วิศวกร |
| วิภา | 30 | นักการตลาด |

## Footnotes

### การสร้าง Footnote

ใช้ `[^1]` สำหรับอ้างอิง และ `[^1]:` สำหรับนิยาม:

```markdown
นี่คือข้อความที่มี footnote[^1]

นี่คือข้อความที่มี footnote อีก[^2]

[^1]: นี่คือเนื้อหาของ footnote แรก
[^2]: นี่คือเนื้อหาของ footnote ที่สอง
```

### Footnote พร้อมรูปแบบ

```markdown
md2docx รองรับภาษาไทย[^th] และภาษาอังกฤษ[^en]

[^th]: รองรับฟอนต์ไทยมาตรฐาน เช่น TH Sarabun New
[^en]: รองรับฟอนต์อังกฤษ เช่น Calibri, Arial
```

### ตัวอย่าง Footnote ภาษาไทย

```markdown
# การอ้างอิง

md2docx เป็นเครื่องมือที่พัฒนาด้วยภาษา Rust[^rust]
ซึ่งเป็นภาษาที่มีความปลอดภัยและมีประสิทธิภาพสูง[^performance]

## ประวัติความเป็นมา

โครงการนี้เริ่มต้นในปี 2568[^start]
โดยมีเป้าหมายเพื่อแก้ปัญหาการแปลง Markdown เป็น DOCX[^problem]

[^rust]: Rust เป็นภาษาโปรแกรมที่พัฒนาโดย Mozilla
[^performance]: Rust มีประสิทธิภาพสูงเทียบเท่ากับ C++
[^start]: โครงการเริ่มต้นในเดือนมกราคม 2568
[^problem]: ปัญหาหลักคือการแสดงผลภาษาไทยที่ไม่ถูกต้อง
```

## Frontmatter (YAML)

Frontmatter คือข้อมูลเมตาดาตาที่อยู่ด้านบนสุดของไฟล์ Markdown ใช้ `---` คร่อม:

### Frontmatter พื้นฐาน

```markdown
---
title: "ชื่อเอกสาร"
author: "ผู้แต่ง"
date: "2025-01-26"
---

# เนื้อหาเอกสาร
```

### Frontmatter สำหรับ md2docx

```markdown
---
title: "บทนำ"
title_th: "บทนำ"
skip_toc: false
skip_numbering: false
page_break_before: true
---

# บทนำ
```

### ตัวเลือก Frontmatter

| ตัวเลือก | คำอธิบาย | ค่าเริ่มต้น |
|---------|-----------|-------------|
| `title` | ชื่อเอกสาร/บท | - |
| `title_th` | ชื่อภาษาไทย | - |
| `skip_toc` | ข้ามสารบัญ | `false` |
| `skip_numbering` | ข้ามการลำดับหน้า | `false` |
| `page_break_before` | เริ่มหน้าใหม่ก่อน | `true` |

### ตัวอย่าง Frontmatter ภาษาไทย

```markdown
---
title: "การติดตั้ง"
title_th: "การติดตั้ง"
skip_toc: false
skip_numbering: false
page_break_before: true
---

# การติดตั้ง

บทนี้จะอธิบายวิธีการติดตั้ง md2docx บนระบบปฏิบัติการต่างๆ
```

## สรุปบทนี้

ในบทนี้คุณได้เรียนรู้รูปแบบการเขียน Markdown ทั้งหมด:

- **หัวข้อ** - ใช้ `#` สำหรับระดับต่างๆ
- **ย่อหน้า** - ข้อความธรรมดา
- **การจัดรูปแบบ** - ตัวหนา ตัวเอียง ขีดฆ่า โค้ด
- **รายการ** - แบบไม่เรียงลำดับและแบบเรียงลำดับ
- **ลิงก์** - ลิงก์ภายในและภายนอก
- **รูปภาพ** - รูปภาพพร้อมขนาดและ caption
- **ตาราง** - ตารางพร้อมการจัดแนว
- **Code blocks** - โค้ดพร้อม syntax highlighting
- **Blockquotes** - ข้อความอ้างอิง
- **เส้นแบ่ง** - แบ่งส่วนเอกสาร
- **Footnotes** - เชิงอรรถ
- **Frontmatter** - ข้อมูลเมตาดาตา
